**Unified Timeline section** replacing the old "Processing Progress" + "Timeline" + "Failure Box":

Create a `buildTimeline()` function that takes `data.purchase` and `data.processingLogs` and returns an array of timeline events:

```typescript
interface TimelineEvent {
  key: string
  label: string
  timestamp: string | null // ISO string or null
  status: 'done' | 'active' | 'pending' | 'error'
  detail?: string // error message, refund reason, etc.
  detailExpanded?: string // failureDetails JSON
}
```

Logic for buildTimeline:
1. Always add: `{ key: 'pending', label: 'Compra criada', timestamp: purchase.createdAt, status: 'done' }`
2. If `purchase.paidAt`: `{ key: 'paid', label: 'Pagamento confirmado', timestamp: purchase.paidAt, status: 'done' }`
3. If status is PROCESSING, COMPLETED, FAILED, or REFUNDED — add pipeline steps from `processingLogs`:
   - For each log: map `log.status === 'completed'` to `'done'`, `'in_progress'` to `'active'`, `'pending'` to `'pending'`
   - Special case: if `purchase.status === 'FAILED'` and `log.step === purchase.processingStep` and `log.status !== 'completed'`, set status to `'error'`
   - Special case: if `purchase.status === 'COMPLETED'`, all steps are `'done'`
4. If status is COMPLETED: `{ key: 'completed', label: 'Relatorio gerado', timestamp: purchase.updatedAt, status: 'done' }`
5. If status is FAILED: `{ key: 'failed', label: getFailureMessage(purchase.failureReason), timestamp: purchase.updatedAt, status: 'error', detail: purchase.failureReason !== purchase.failureDetails ? purchase.failureDetails : undefined, detailExpanded: purchase.failureDetails }`
6. If status is REFUNDED: `{ key: 'refunded', label: 'Reembolso processado', timestamp: purchase.updatedAt, status: 'done', detail: getRefundMessage(purchase.refundReason) }`

**Timeline visual rendering** (vertical, top-to-bottom):
