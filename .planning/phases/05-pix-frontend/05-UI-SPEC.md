Declared values (all multiples of 4 — from `--primitive-space-*` tokens):

| Token | Value | CSS Variable | Usage |
|-------|-------|--------------|-------|
| xs | 4px | `--primitive-space-1` | Icon gaps, countdown digit gap |

| lg | 24px | `--primitive-space-6` | Card padding, section spacing |

- QR code image: fixed 200x200px (dictated by AbacatePay PNG dimensions)
- Touch targets (copy button, renew button): minimum 44px height per mobile accessibility requirement
- Countdown timer: `--primitive-space-3` (12px) vertical padding to maintain visual weight

## Typography

All roles use existing token system. Two font families: Zilla Slab (headings), IBM Plex Mono (body/UI).

Four sizes total (maximum allowed):

| Role | Size | Weight | Line Height | Font | CSS Token |
|------|------|--------|-------------|------|-----------|
| Body | 14px (0.875rem) | 400 | 1.7 | IBM Plex Mono | `--font-body` |
| Label | 11px (0.6875rem) | 700 | 1.0 | IBM Plex Mono | `--font-label` |
| Heading (page title) | 28px (1.75rem) | 700 | 1.1 | Zilla Slab | `--font-h2` |
| Display (countdown timer) | 20px (1.25rem) | 700 | 1.0 | IBM Plex Mono | `--primitive-size-xl` + bold |

Typography rules for this phase:
- Countdown MM:SS display: IBM Plex Mono, 20px, weight 700, monospace tabular numbers — use `font-variant-numeric: tabular-nums` to prevent layout shift
- Pix code (brCode text): IBM Plex Mono, 11px, weight 400, word-break: break-all — the Pix EMV code is 80+ characters
- All badge/status text: 11px, uppercase, letter-spacing 0.125em (per `--primitive-tracking-wider`)
- Body copy (instruction text): IBM Plex Mono 14px, weight 400, line-height 1.7

Source: `tailwind.config.ts` fontSize scale, `tokens.css` `--font-*` semantic tokens

Accent (`#FFD600`) reserved for:
1. Primary CTA button background ("COPIAR CODIGO PIX", "GERAR NOVO QR CODE")
2. Badge label background ("AGUARDANDO PAGAMENTO", "PIX EXPIRADO")
3. Countdown timer border/underline when time is above 60 seconds
4. Focus ring on interactive elements (`--input-shadow-focus`)

Destructive (`#CC3333`) reserved for:
1. Expired state banner background tint (`--primitive-red-100` = #FFF0F0)
2. Expired state icon and border-left accent
3. Error state (network error, API error) text and callout

Countdown urgency rule: when `secondsLeft <= 60`, countdown text color transitions from `--color-text-primary` (#1A1A1A) to `--primitive-red-500` (#CC3333) to signal urgency.

## Interaction Contract

### State Machine

```
loading → waiting    (POST /api/purchases/pix succeeds, expiresAt is in the future)
loading → expired    (POST succeeds but expiresAt already passed on load)
loading → error      (POST fails or returns 4xx/5xx)
waiting → paid       (poll returns PAID or COMPLETED — redirect to /minhas-consultas)
waiting → expired    (countdown reaches 0, or poll returns EXPIRED)
expired → loading    (user clicks "GERAR NOVO QR CODE")
error   → loading    (user clicks "Tentar novamente")
```

### Polling Behavior

- Interval: 3 seconds (`setInterval` 3000ms)
- Active only when `pixState === 'waiting'`
- Cleanup: `clearInterval` on every state transition and on component unmount
- Network errors: silent catch — next poll retries automatically
- On PAID/COMPLETED: set `pixState = 'paid'` first, then `router.push('/minhas-consultas')` — prevents interval from firing during navigation

### Countdown Timer

- Tick: 1 second (`setInterval` 1000ms)
- Active only when `pixState === 'waiting'`
- Cleanup: `clearInterval` on every state transition
- Display format: `MM:SS` (e.g., "29:47") — zero-padded with `String(n).padStart(2, '0')`
- On `secondsLeft <= 0`: immediately set `pixState = 'expired'` without waiting for next poll
- Color change: `secondsLeft <= 60` → text becomes `--primitive-red-500`

### Copy to Clipboard

- Action: `navigator.clipboard.writeText(brCode)`
- Button label: "COPIAR CODIGO PIX"
- Feedback: button label changes to "COPIADO!" for 2000ms, then reverts
- Button state during feedback: disabled (prevents double-copy)
- Accessibility: `aria-live="polite"` on the button label span so screen readers announce "COPIADO!"

### Mock Mode

When `brCodeBase64.startsWith('data:image/png;base64,BYPASS')`:
- Do not render `<img>` tag (broken image)
- Render a grey placeholder box: 200x200px, `background: --primitive-gray-100`, dashed border `2px dashed --primitive-gray-300`
- Overlay text in center: "QR Code (Mock)" in 11px IBM Plex Mono, `--color-text-muted`
- Copy button still works with the mock `brCode` value

### Entry Point Update

`consulta/[term]/page.tsx` must redirect to `/compra/pix?purchaseId=<uuid>` instead of `/compra/confirmacao?code=<code>` after a successful purchase creation. The `purchaseId` is the UUID returned in the `POST /api/purchases` response, not the short `code`.

## Button Styles

All buttons follow the existing BEM class pattern from `styles/index.css`:

| Variant | Class | Usage |
|---------|-------|-------|
| CTA (primary) | `btn btn--cta btn--full` | "COPIAR CODIGO PIX", "GERAR NOVO QR CODE", "TENTAR NOVAMENTE" |
| Ghost | `btn btn--ghost` | "Tentar novamente" text link variant in error state |

CTA visual contract (from `--btn-cta-*` tokens):
- Background: `#FFD600`
- Text: `#1A1A1A`, weight 700, 14px IBM Plex Mono, uppercase
- Border: `2px solid #1A1A1A`
- Box shadow: `3px 3px 0 0 #1A1A1A` (brutalist offset)
- Hover: shadow becomes `4px 4px 0 0 #1A1A1A`
- Active: shadow becomes `1px 1px 0 0 #1A1A1A`
- Disabled: opacity 0.5, cursor not-allowed
