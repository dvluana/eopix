**1. src/components/PixCheckout.tsx** — "use client" component implementing the PIX state machine.

State type:
```typescript
type PixState = 'loading' | 'waiting' | 'expired' | 'paid' | 'error'
```

Interface:
```typescript
interface PixData {
  pixId: string
  brCode: string
  brCodeBase64: string
  expiresAt: string
}
export default function PixCheckout({ purchaseId }: { purchaseId: string })
```

Implementation details:
- **Loading state (initial):** Call `POST /api/purchases/pix` with `{ purchaseId }`. On success, check if `expiresAt` is already past — if so go to `expired`, else go to `waiting` with pixData. On error go to `error`. Show `<EopixLoader text="Gerando seu codigo PIX..." />`.

- **Waiting state:** Render the QR code card + countdown + copy button + trust line.
  - QR Code: `<img src={pixData.brCodeBase64} alt="QR Code PIX" width={200} height={200} className="pix-qr-card__img" />` with `image-rendering: pixelated`
  - Mock mode detection: if `brCodeBase64.includes('BYPASS')`, render grey placeholder div (200x200, dashed border, text "QR Code (Mock)") instead of `<img>`.
  - Pix code block: dark background `<div className="pix-code-block">` showing the `brCode` text with `word-break: break-all` and `user-select: all`.
  - Copy button: `btn btn--cta btn--full` calling `navigator.clipboard.writeText(brCode)`. Label "COPIAR CODIGO PIX", changes to "COPIADO!" for 2000ms. Disabled during feedback. Add `aria-live="polite"` on label span.
  - Countdown: `useEffect` with 1-second `setInterval`. Calculate `secondsLeft = Math.max(0, Math.floor((new Date(expiresAt).getTime() - Date.now()) / 1000))`. Display as `MM:SS` with `String(m).padStart(2,'0'):String(s).padStart(2,'0')`. Use `font-variant-numeric: tabular-nums`. When `secondsLeft <= 60`, add class `pix-countdown--urgent` (red text). When `secondsLeft <= 0`, set `pixState = 'expired'`.
  - Status polling: `useEffect` with 3-second `setInterval` (only when `pixState === 'waiting'`). `GET /api/purchases/pix/status?purchaseId=${purchaseId}`. On `PAID` or `COMPLETED`: set `pixState = 'paid'` then `router.push('/minhas-consultas')`. On `EXPIRED`: set `pixState = 'expired'`. Network errors: silent catch. Always `clearInterval` on cleanup and state change.
  - Status badge: `<span className="pix-badge pix-badge--waiting">AGUARDANDO PAGAMENTO</span>`
  - Trust line: `<p className="pix-trust">Pagamento 100% seguro · Dados criptografados · PIX via AbacatePay</p>`

- **Expired state:** Replace QR card with expired callout.
  - Badge: `<span className="pix-badge pix-badge--expired">PIX EXPIRADO</span>`
  - Callout: red-tinted div with 3px red left border. Heading "Codigo PIX expirado", body "Este codigo PIX expirou. Gere um novo para continuar o pagamento."
  - Renew button: `btn btn--cta btn--full` "GERAR NOVO QR CODE" — on click, set `pixState = 'loading'` and call `POST /api/purchases/pix` again (per D-08, backend overwrites expired charge). Use a `renewKey` counter state to re-trigger the init useEffect.

- **Paid state:** Brief "Pagamento confirmado! Redirecionando..." text before redirect completes.

- **Error state:** Red callout with heading "Nao foi possivel gerar o PIX", body "Verifique sua conexao e tente novamente.", CTA button "TENTAR NOVAMENTE" which sets `pixState = 'loading'` (retriggers init).

Use proper Portuguese accents in copy (e.g., "codigo" -> "codigo" — follow existing pattern in confirmacao page).
